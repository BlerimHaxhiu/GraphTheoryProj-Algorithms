# Educational Value

This project is designed as an educational technology tool for learning graph algorithms through interaction, visualization, and guided explanation.

## Why Algorithm Visualization Matters

Graph algorithms are abstract. Students often see pseudocode, final results, or static diagrams, but they do not always see how the algorithm makes decisions.

Visualization makes invisible state visible:

- which node is being visited,
- which edge is being considered,
- which path is currently best,
- how a matrix changes,
- why one algorithm behaves differently from another.

This helps learners connect theory to behavior.

## Why Step-By-Step Execution Matters

Step-by-step execution slows the algorithm down enough for learners to reason about it.

Instead of only seeing an answer, students can observe:

- initialization,
- traversal,
- relaxation,
- path reconstruction,
- edge selection,
- cycle rejection,
- completion conditions.

This is especially valuable for algorithms such as Dijkstra, Bellman-Ford, Floyd-Warshall, Kruskal, and Prim, where intermediate state is the key to understanding.

## Why Bilingual Support Matters

Students learn technical concepts more effectively when explanations are available in a language they understand well.

English support helps with global technical vocabulary. Albanian support makes the tool more accessible for Albanian-speaking learners and classrooms.

Bilingual support also gives instructors flexibility:

- teach terminology in English,
- explain concepts in Albanian,
- compare both languages during lessons,
- support students with different comfort levels.

## Why Comparison Mode Matters

Algorithms are not learned in isolation. Students need to understand tradeoffs:

- BFS vs DFS,
- Dijkstra vs Bellman-Ford,
- Dijkstra vs A*,
- Kruskal vs Prim,
- Floyd-Warshall vs single-source shortest path algorithms.

Comparison mode helps students see that algorithm choice depends on graph structure, edge weights, directedness, and the question being asked.

## Why Chatbot Guidance Matters

The chatbot provides a low-friction way for students to ask questions while staying inside the learning environment.

It can:

- explain algorithms,
- define graph terms,
- compare approaches,
- describe the current step,
- warn about common mistakes,
- run supported commands through natural language.

This creates a guided learning experience rather than a passive simulator.

## Why Graph Editing Matters

Graph editing lets students experiment.

They can change:

- edge weights,
- graph direction,
- graph density,
- start/end nodes,
- graph shape,
- cycles,
- connectivity.

This experimentation is important because graph algorithms are sensitive to structure. A small graph change can produce a different traversal, shortest path, or spanning tree.

## Value For Educators

Educators can use the platform to:

- demonstrate algorithms live,
- ask students to predict the next step,
- compare algorithms on the same graph,
- show matrix and graph representations together,
- export graphs and reports for assignments or presentations,
- support bilingual instruction.

The platform is strongest as a teaching aid, lab companion, and portfolio-quality educational software project.
