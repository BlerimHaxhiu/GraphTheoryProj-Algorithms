# Project Positioning

This project should be positioned as an interactive graph algorithm visualization and learning platform, not as a simple university graph visualizer.

## Compared With A Typical University Graph Visualizer

A typical university graph visualizer often includes:

- a small number of algorithms,
- basic node/edge drawing,
- final result display,
- limited explanation,
- no localization,
- little or no testing,
- no export/reporting flow,
- minimal repository documentation.

This repository is stronger because it includes:

- 8 supported graph algorithms,
- interactive SVG graph editing,
- weighted, directed, undirected, curved, parallel, and self-loop edges,
- step-by-step algorithm visualization,
- adjacency matrix display,
- algorithm report panels,
- graph statistics,
- execution history,
- algorithm comparison,
- JSON export,
- PNG export,
- printable reports,
- English and Albanian localization,
- educational chatbot assistance,
- parser tests,
- algorithm tests,
- translation parity tests,
- professional documentation and audit artifacts.

## Relevance For Software Engineering

The project demonstrates:

- data modeling for graphs,
- separation between algorithm logic and UI rendering,
- stateful application architecture,
- validation flows,
- import/export workflows,
- regression testing,
- documentation discipline,
- maintainability awareness through documented limitations.

## Relevance For Frontend Engineering

The project demonstrates:

- complex React state management,
- custom SVG interaction,
- drag/pan/edit workflows,
- responsive dashboard layout,
- dialogs and panels,
- theming,
- localization,
- visual feedback and animation,
- user-driven data visualization.

## Relevance For Educational Technology

The project demonstrates:

- interactive learning design,
- step-by-step concept explanation,
- bilingual accessibility,
- learning reports,
- algorithm comparison,
- instructor-friendly exports,
- support for experimentation and classroom demonstration.

## Relevance For AI Automation Engineering

The current chatbot is deterministic, but it shows automation thinking:

- natural language command parsing,
- intent detection,
- controlled action execution,
- validation before automation,
- bilingual command support,
- educational response generation.

This creates a safe foundation for future AI integration where a model could provide richer tutoring while the existing deterministic action layer remains responsible for validated app actions.

## Recommended Portfolio Framing

Use this framing:

> A bilingual interactive graph algorithm learning platform with custom SVG graph editing, step-by-step algorithm visualization, reports, exports, comparison tools, and a deterministic chatbot command layer.

Avoid this framing:

> A graph theory homework simulator.

The stronger story is that this is a multi-surface educational software system with algorithmic depth, frontend complexity, localization, automation thinking, and practical documentation.
